1. ¿Qué es un Dato?

Un dato es una representación simbólica de un atributo o característica de una entidad. Puede ser cualquier hecho, observación o circunstancia que tenga un significado relativo dentro de un contexto determinado.

2. ¿Qué es información?

La información es el resultado del procesamiento, manipulación y organización de datos de manera que adquieren significado y relevancia para quien la utiliza. Es decir, los datos se transforman en información cuando se interpretan y se les atribuye un contexto y sentido.

3. ¿Diferencia entre dato e información?

La diferencia fundamental radica en que los datos son la materia prima, mientras que la información es el resultado del análisis y procesamiento de esos datos.

4. ¿Tipos de datos?

Los datos pueden clasificarse en diferentes categorías según su naturaleza, como datos numéricos, datos textuales, datos categóricos, datos temporales, etc.

5. ¿Qué es un modelado de Datos?

El modelado de datos es el proceso de definir y organizar la estructura de una base de datos, representando las entidades, atributos, relaciones y restricciones que la componen.

7. ¿Qué es el modelo relacional?

El modelo relacional es un modelo de datos que organiza la información en tablas (relaciones) que se relacionan entre sí mediante claves primarias y claves foráneas.

8. ¿Describa los elementos del Diagrama entidad Relacional?

Los elementos principales son las entidades, los atributos y las relaciones. Las entidades representan objetos del mundo real, los atributos son las características de esas entidades y las relaciones muestran cómo están relacionadas entre sí las entidades.

9. ¿Qué es una entidad?

Una entidad es un objeto o concepto del mundo real que se puede distinguir claramente y del cual se pueden recolectar datos. Por ejemplo, una entidad podría ser un cliente, un producto o una orden de compra.

10. ¿Qué es un atributo y sus tipos?

Un atributo es una propiedad o característica que describe una entidad. Puede ser de diferentes tipos, como atributos numéricos, textuales, de fecha, booleanos, etc.

11. ¿Qué es una relación?

Una relación es una asociación entre dos o más entidades que representa una conexión significativa entre ellas. Por ejemplo, la relación "compra" entre un cliente y un producto en una base de datos de ventas.

12. ¿Qué son las llaves primarias, foránea y sus características?

La llave primaria es un atributo o conjunto de atributos que identifica de manera única a cada registro en una tabla. La llave foránea es un atributo o conjunto de atributos que establece una relación entre dos tablas, vinculando la llave primaria de una tabla con la llave primaria o secundaria de otra tabla.

13. ¿Qué son las reglas del Negocio?

Las reglas del negocio son directrices o restricciones que definen cómo debe operar una organización o sistema. Estas reglas pueden influir en el diseño y funcionamiento de una base de datos.

14. ¿Qué es la cardinalidad o mapeo?

La cardinalidad o mapeo en el contexto de las bases de datos describe la relación entre entidades y cómo se relacionan unas con otras. Por ejemplo, la cardinalidad uno a uno, uno a muchos, muchos a muchos, etc., describe cuántas instancias de una entidad pueden relacionarse con cuántas instancias de otra entidad.
